#第一个项目来源于-UCI Online Retail数据集https://archive.ics.uci.edu/dataset/352/online+retail
核心工作：
1.数据清洗与建模：使用 Python 对原始交易数据进行清洗（处理缺失值、异常值、退货记录），并将数据拆分为订单主表、订单明细表、客户表、
3.核心指标分析：计算总销售额、总订单数、平均客单价、复购率、交易趋势变化等核心业务指标，通过 Tableau 构建交互式仪表盘，实现业务指标的实时监控与可视化。
3.用户价值分层：用python基于 RFM 模型对已知客户进行价值分层（高价值 / 中高价值 / 中价值 / 低价值），识别核心用户群体，建立可视化分层。
这个数据集有50w条数据，我电脑性能不行用tableau处理大批量数据会卡所以就建了三张聚合表方便可视化展示，在数据表里建立索引可以再次优化查询数据的速度，想深入分析可以用一些机器学习和统计学的知识，由于本人技能水平不强所以就只能暂时做到这种地步。

第二个项目来源于- UCI客户流失数据集：https://archive.ics.uci.edu/dataset/563/iranian+churn+dataset
核心工作：
1.数据层：使用 Navicat 管理 MySQL 数据库，设计并维护客户行为表、特征重要性表、相关性矩阵表等结构化数据，保障数据存储规范与查询效率；
2.分析层：通过 Python 完成数据清洗（缺失值 / 异常值处理）、特征工程（相关性分析、特征筛选），输出特征重要性排序及高风险客户判定规则（满足投诉 / 短订阅期 / 高通话失败率 / 低客户价值中≥2 项为高风险）；
3.可视化层：使用 Tableau 连接数据库，构建交互式分析看板，包含特征重要性条形图、特征相关性热力图、高低风险客户流失率对比图等，直观呈现 “投诉、使用时长、使用频率” 为流失核心驱动因子，及高风险客户流失率（45.22%）显著高于低风险客户（7.84%）的核心结论。
这是个轻量化的数据集才3000条，可以用来熟悉tableau和初步了解数据分析。
